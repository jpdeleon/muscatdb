Configure & Run TTV Fitting

Transit timing data is derived from the O-C (Observed minus Calculated) plot above. Configure harmonic model parameters and run the TTV fit.

Model Options
INI Configuration ?

Auto-generated from the planet letters above. Edit directly to override the initial guesses — harmonic samples around them, so they only need to be the right order of magnitude.

[INIT] — required. One group per adjacent pair of planets, named after the pair: letters bcd give the pairs bc and cd. For pair bc:

  • a_bc, a_cb — TTV amplitude each planet induces on the other, in days (default 0.02).
  • per_bc — TTV super-period, in days (default 1000).
  • t_bc — TTV phase reference, in BJD (seeded from the first planet's T0, else 2450000).

At least one a_* amplitude guess must be present or the run is rejected before launch.

[T14] — optional. Transit durations in days, as t14_b, t14_c, … This section is rebuilt from the Transit Duration T14 fields in Planetary Reference Ephemerides above every time you edit one, so change a duration there rather than here. Planets with a blank or zero duration are omitted. Rebuilding preserves your [INIT] and [OUTER] edits.

[OUTER] — optional. Parameters for the non-transiting outer planet. Not auto-generated: add it yourself when Non-transiting outer planet is checked. See Kepler-51 example

Changing the planet letters regenerates the whole file from defaults and discards manual edits, including [OUTER].
